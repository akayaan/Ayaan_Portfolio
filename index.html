ABOUT

Hello, I'm Ayaan Khan, and this is my portfolio showcasing my work as a Unity Developer, Technical Artist, and Game Programmer.

I am currently pursuing an M.Sc. in Game Technology, where I am building strong skills in game design, development, and technical art. My goal is to become a Game Technical Artist and Game Developer, combining creativity with technical expertise to craft engaging and visually immersive experiences. I enjoy exploring the balance between art and technology—whether it's bringing characters and environments to life, optimizing workflows, or experimenting with new game mechanics. Driven and curious, I am eager to contribute to the gaming industry with innovative ideas and impactful projects.

Outside of academics and game development, I enjoy watching anime, drawing, exploring new places, and trying out different kinds of food. I also have a keen interest in learning new languages, especially Japanese, as I love discovering new cultures and perspectives.

SUPERMARKET SHOWDOWN

Supermarket Showdown is a hilarious ragdoll PVP game set in a chaotic supermarket environment. Both players have the same shopping list and race to collect all the items first. Along the way, they can create chaos by throwing products, spilling oil, or stealing from each other's cart. With silly physics and funny interactions, the game turns everyday shopping into a wild and competitive experience that's perfect for laughs and friendly battles.

Project Summary

Project Type Physics Based Ragdoll
Focus PVP Party Game, Ragdoll Physics, Multiplayer Chaos
Engine Unity
Game Type PVP Party Game
My Role Gameplay Programmer & Physics Systems Developer

Technical Contributions

  • Ragdoll Physics System: Developed the core ragdoll physics mechanics that create hilarious character movements and interactions
  • Multiplayer PVP Systems: Implemented real-time multiplayer functionality for competitive shopping chaos
  • Inventory & Shopping System: Created the shopping list mechanics and inventory management for both players
  • Interactive Environment: Designed and implemented interactive supermarket elements like slippery oil spills and throwable products
  • Cart Mechanics: Developed the shopping cart physics and stealing mechanics for competitive gameplay

Game Features

Ragdoll Chaos

Hilarious physics-based character movements and collisions

Competitive Shopping

Race against another player to complete your shopping list first

Silly Sabotage

Throw products, spill oil, and steal items to disrupt your opponent

DRIFT RUSH

Drift Rush is an exciting endless runner prototype built in Unity that focuses on high-speed drifting mechanics and dynamic obstacle navigation. Players control a vehicle that must maintain speed while performing precision drifts to avoid obstacles and collect power-ups.

Project Summary

Project Type Prototype
Focus Drifting Mechanics, Dynamic Obstacle Generation, Speed Management
Engine Unity (C#)
My Role Solo Developer (Designed and implemented all game systems)

Technical Contributions

  • Drifting Physics: Implemented realistic drifting mechanics with proper vehicle physics and traction control.
  • Procedural Generation: Developed a system that dynamically creates challenging track layouts with increasing difficulty.
  • Obstacle System: Created varied obstacle patterns that require precise timing and drifting to avoid.
  • Score Multiplier: Designed a scoring system that rewards players for maintaining drift chains and speed.

WEAPON THROW & RECALL SYSTEM

This prototype is a technical demonstration inspired by the impactful combat and visual fidelity of God of War (2018). The project showcases a real-time system for simulating deep, clean slices or cuts into a large surface upon weapon impact. This effect is achieved through custom shaders or mesh manipulation techniques, ensuring the slice mark is instantly and precisely drawn onto the geometry where the weapon hits. It highlights proficiency in creating responsive, high-fidelity combat visuals for games.

Project Summary

Project Type Prototype
Focus Weapon Mechanics, Physics-based Throwing, Recall System, Combat Visuals
Engine Unity (C#)
Inspiration God of War (2018) combat mechanics
My Role Solo Developer (Implemented all weapon systems and visual effects)

Technical Contributions

  • Weapon Throw Mechanics: Developed physics-based throwing system with realistic trajectory and impact physics
  • Recall System: Implemented smooth weapon recall mechanics with visual trail effects and precise return pathing
  • Dynamic Impact Effects: Created real-time slicing and cutting effects on surfaces using custom shaders
  • Combat Integration: Seamlessly integrated throw/recall mechanics with core combat systems
  • Visual Feedback: Designed particle systems and visual effects for weapon trajectory and impacts

System Features

Physics-based Throwing

Realistic weapon trajectory with gravity and collision detection

Smart Recall

Intelligent return pathing with obstacle avoidance

Dynamic Damage

Impact-based damage system with environmental destruction

FIRST PERSON SHOOTER

This project is a First-Person Shooter (FPS) prototype built in Unity (C#) to demonstrate essential combat systems. It features a complete Weapon System for multiple guns with full reloading logic and different fire modes. The focus was on using Hitscan/Raycasting to drive dynamic object destruction, allowing targets and barriers to break realistically upon impact. This showcases strong skills in core gameplay programming and responsive combat feedback.

Project Summary

Project Type Prototype / Combat System Demo
Focus Weapon Systems, Hitscan/Raycasting, Dynamic Destruction, Combat Feedback
Engine Unity (C#)
My Role Solo Developer (Designed and implemented all combat systems)

Technical Contributions

  • Weapon System: Developed a complete weapon system supporting multiple guns with different fire modes and full reload mechanics.
  • Hitscan/Raycasting: Implemented precise hit detection using raycasting for realistic bullet trajectories and impact.
  • Dynamic Destruction: Created a system for realistic object destruction where targets and barriers break upon impact.
  • Combat Feedback: Designed responsive visual and audio feedback systems for immersive combat experience.
  • Ammo Management: Programmed comprehensive ammo and reload systems with different magazine capacities.

FUN & EDUCATIONAL PUZZLE

This educational puzzle game combines fun gameplay with learning reasoning skills, inspired by games like Baba Is You. Players solve various types of reasoning puzzles including Coding-Decoding, Blood Relations, and Direction Sense problems. Each level presents a unique challenge where players must manipulate blocks to form correct solutions, unlocking doors and collecting treasures as they progress.

Project Summary

Project Type Prototype / Educational Game
Focus Reasoning Puzzles, Educational Gameplay, Puzzle Mechanics
Engine Unity (C#)
My Role Solo Developer (Designed and implemented all puzzle systems and mechanics)

Technical Contributions

  • Puzzle System: Developed multiple puzzle types including Coding-Decoding, Blood Relations, and Direction Sense challenges
  • Block Manipulation: Created intuitive drag-and-drop mechanics for puzzle solving
  • Progression System: Implemented level progression with treasure collection and door unlocking mechanics
  • Educational Content: Designed puzzles that teach logical reasoning skills while maintaining engaging gameplay
  • UI/UX Design: Created clean, intuitive interfaces that guide players through complex reasoning problems

TRIM & TONIC

Trim & Tonic is a resource management prototype where players work in a cocktail shop. The core gameplay loop involves mixing various drinks to make the perfect drink for our customer, balancing precise pouring mechanics with speed and order complexity.

Project Summary

Project Type Prototype / Core Mechanic Demo
Focus Mixing Logic, Customer Order Handling, UI Feedback
Engine Unity (C#)
My Role Systems Programmer (Developed the ingredient mixing and order validation systems)

Technical Contributions

  • Mixing Logic: Implemented the core system for combining ingredients in specific ratios to meet customer demands.
  • Precision Mechanics: Developed the UI and code for a timing/meter system to ensure accurate "pouring."
  • Customer Satisfaction: Programmed customer queues and a satisfaction/patience rating linked to game scoring.
